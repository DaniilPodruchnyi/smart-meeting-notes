# `internal/adapters/persistence`

## Назначение

Слой доступа к данным: реализация интерфейсов репозиториев для PostgreSQL (и при необходимости миграции рядом или в отдельной папке).

## Подпакеты

- `postgres/` — конкретная реализация на `database/sql` или `pgx`.

## Что реализовать по ТЗ

- Таблицы (на усмотрение команды): пользователи, встречи/записи, полный текст транскрипции, метаданные (дата, telegram file id опционально).
- Сохранение и выдача транскрипции для `/get <id>`.
- `/list`: список встреч пользователя по дате.
- `/find`: поиск по ключевым словам — `LIKE` / полнотекстовый поиск PostgreSQL (`to_tsvector` / GIN) и т.д.

## Правила

- SQL и миграции не протекают в usecase — только методы интерфейсов репозитория.
