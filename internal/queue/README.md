# `internal/queue`

## Назначение

Очередь обработки фоновых задач через **каналы** (`chan`), как рекомендует ТЗ: тяжёлые вызовы SaluteSpeech и GigaChat не должны «замораживать» приём сообщений бота.

## Что реализовать

- Тип задачи: например `{ UserID int64, ChatID int64, Kind enum, Payload ... }` — чтобы ответ ушёл тому же пользователю.
- Один или несколько **воркеров** (`go worker()`), читающих из канала и вызывающих usecase или сервисы.
- Канал результатов обратно к слою Telegram (или callback), который шлёт сообщение в нужный chat.
- Graceful shutdown: закрытие канала после отмены контекста, дождаться воркеров.

## Правила

- Бизнес-логика «что такое транскрипция» остаётся в usecase; очередь только маршрутизирует работу и user id.
