# `internal/app/usecase`

## Назначение

Прикладные сценарии (application layer): оркестрация «что делать», без деталей telebot и без сырого SQL.

## Что реализовать по ТЗ

1. **Регистрация** (`/start`): сохранить пользователя в БД.
2. **Приём аудио/голоса**: координация загрузки файла → вызов SaluteSpeech → сохранение транскрипции → запрос к GigaChat на краткую выжимку → сохранение и ответ пользователю (ответ может идти через очередь).
3. **Список встреч** (`/list`): отсортировать по дате.
4. **Транскрипция по id** (`/get <id>`): проверка прав (встреча этого пользователя).
5. **Поиск** (`/find <сниппет>`): поиск по тексту транскрипций (стратегия — в репозитории).
6. **Чат с ИИ** (`/chat`): сформировать prompt и вызвать GigaChat; при желании — контекст последней встречи из БД.

## Зависимости (через интерфейсы)

- Репозиторий пользователей и встреч / транскрипций.
- Порты (интерфейсы) для `SaluteSpeech` и `GigaChat`.
- Доступ к очереди задач (постановка задачи «транскрибировать» / «саммари» / «chat»).

## Правила

- Не импортировать `gopkg.in/telebot.v3` напрямую — только типы данных, которые передаёт adapter.
