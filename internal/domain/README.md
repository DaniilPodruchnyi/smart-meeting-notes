# `internal/domain`

## Назначение

Доменные типы без зависимостей от Telegram, HTTP-клиентов и БД-драйвера: сущности и при необходимости доменные ошибки.

## Что реализовать (ориентир по ТЗ)

- Пользователь бота (Telegram user id, дата регистрации).
- Встреча / запись: идентификатор, связь с пользователем, время создания, опционально длительность/имя файла.
- Транскрипция: полный текст, ссылка на краткую выжимку (или хранить выжимку отдельным полем).
- Типы для результатов поиска (например, snippet или список id).

## Правила

- Здесь нет тегов `json` для API Telegram, если не нужно — или держите DTO отдельно в adapters.
- Интерфейсы репозиториев можно объявить рядом с usecase или здесь — главное, чтобы адаптер `postgres` их реализовывал.
