# `internal/server`

## Назначение

Запуск HTTP-сервера приложения (health-check, при необходимости — webhook Telegram).

## Что реализовать

- Поднятие `http.Server` с роутером из `transport/http`.
- Корректная остановка по `context` (уже заложено в каркасе).
- При использовании **webhook**: маршрут `POST` для приёма обновлений от Telegram (проверка секрета, передача в telebot или разбор JSON — по выбранной схеме).

## Правила

- Маршруты не содержат SQL; только вызовы usecase или передача в telegram-adapter.
