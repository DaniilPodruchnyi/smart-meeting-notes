# `internal/server/transport/http`

## Назначение

HTTP-транспорт: маршруты, JSON-ответы, коды статусов.

## Что реализовать сейчас / далее

- Уже есть тестовая ручка `GET /ping`.
- Дополнительно по мере надобности:
  - `GET /health` или `/ready` (проверка БД);
  - `POST` webhook Telegram (если откажетесь от long polling).

## Правила

- Handlers тонкие: парсинг запроса → вызов usecase → формат ответа.
